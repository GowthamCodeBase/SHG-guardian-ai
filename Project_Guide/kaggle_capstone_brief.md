This brief details the rules, tracks, submission requirements, and grading criteria for the **AI Agents: Intensive Vibe Coding Capstone Project**.

## 🎯 Tracks & Awards

### 1. Agents for Good
* **Objective**: Solve problems for humanity (e.g., agriculture optimization, managing public health, advancing education, supporting art and literature).
* **Awards**: Top 3 winners receive Kaggle swag.

### 2. Agents for Business
* **Objective**: Create agents designed to solve compelling business problems with cost or revenue impact on the line (e.g., expense auditing, pipeline action recommendations, commercial insights).
* **Awards**: Top 3 winners receive Kaggle swag.

### 3. Concierge Agents
* **Objective**: Solve individual, family, or social challenges in a safe and secure way that frees up personal time (e.g., invite list managers, medication trackers, garden planners).
* **Awards**: Top 3 winners receive Kaggle swag.

### 4. Freestyle (Open Track)
* **Objective**: Innovative agents that don't fit into neat categories (e.g., satellite tracking, concert recording cataloging, handwriting decoders).
* **Awards**: Top 3 winners receive Kaggle swag.

---

## 📝 Submission Requirements

A valid submission must contain:
1. **Kaggle Writeup**: A detailed project report (limit: 2,500 words).
2. **Media Gallery**: Images and/or cover image demonstrating your application.
3. **YouTube Demonstration Video**: A video of 5 minutes or less uploaded to YouTube demonstrating:
   * Problem Statement: What problem are you solving and why?
   * Agents: Why did you choose an agentic approach to solve this?
   * Architecture: Diagram and description of the system workflow.
   * Live Demo: Interactive walkthrough of the agent in action.
   * Build Process: Explanation of the code, tools, and technologies used.
4. **Public Project Link**: Working interactive demo URL (e.g. Hugging Face Spaces link) or public codebase repository (e.g. GitHub link) containing detailed configuration and setup guides.

## 📊 Evaluation Metrics (100 Points Total)

### Category 1: The Pitch (30 Points)
* **Core Concept & Value (10 pts)**: Clear problem statement, track alignment, novelty, and active agent-centric design.
* **YouTube Video Submission (10 pts)**: Quality of explanation, conciseness (under 5 mins), clear workflow demo, and architectural overview.
* **Writeup Quality (10 pts)**: Readability, documentation of design path, and implementation journey.

### Category 2: Technical Design & Code (70 Points)
* **Technical Implementation (50 pts)**:
  * Overall architecture quality and code readability.
  * Integration of agent tools/skills.
  * Pertinent comments explaining design patterns.
  * Application of at least **three (3) key concepts** learned in the course:
    * *Agent / Multi-agent system (ADK)* (demonstrated in Code)
    * *MCP Server* (demonstrated in Code)
    * *Antigravity* (demonstrated in Video)
    * *Security features* (demonstrated in Code/Video)
    * *Deployability* (demonstrated in Video)
    * *Agent skills / CLI tools* (demonstrated in Code/Video)
  * 🚨 **Important Rule**: Do not check-in any passwords, credentials, or API keys in the code repository.
* **Documentation (20 pts)**: Public GitHub README explaining setup, setup commands, environment configs, and architecture diagrams.
